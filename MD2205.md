## What now works that didn't before

1. **Bulk upload accepts files up to 10 GB each.** Project + bucket file_size_limit both bumped to `10737418240` bytes.
2. **TUS resumable upload** via `tus-js-client@4.3.1`. Streams in 6 MB chunks with retry; survives network blips, tab backgrounding, the moov-atom-at-end stall on phone-camera MP4s.
3. **Real per-file progress** ‚Äî `uploading 35%` climbs in 5% steps. No more "did this freeze?" UX.
4. **Background upload queue** ‚Äî module-level `uploadQueue` singleton. Click "Queue N for upload" and the modal closes immediately. Navigation, tab switches, and additional modal opens don't kill in-flight uploads.
5. **Floating UploadDock** (bottom-right) ‚Äî compact pill with aggregate progress; click to expand into per-file list with progress bars, error messages, dismiss buttons.
6. **Auto-refresh on idle** ‚Äî when the queue empties, library list re-fetches in background mode so new rows surface without a manual refresh.
7. **Claude Vision actor identification** ‚Äî `identify-actor` Edge Function sends each thumbnail + reference frames to Haiku 4.5 Vision; writes `creator` + `visual_description` when confidence ‚â• 0.7. Brand-funnel labels (MAKEUGC, PLUMBERS, TRADIES, ELECTRICIANS, ROOFERS, PROJECT2, UNK) excluded from references.
8. **Auto canonical naming** ‚Äî `creative-library-describe` builds `<PREFIX>-<CREATOR>-<TOPIC>-T01.mp4` from the row's creator + transcript (videos) OR visual_description (images).
9. **NotificationBell no longer crashes the page** ‚Äî `pendingApproval` is now derived from submissions inline (was undefined ref).

## Bug fixes from code-review pass

These were caught by the proactive-code-reviewer after the initial `14e68ac` ship:

1. **NotificationBell `pendingApproval is not defined`** ‚Äî page-crashing ReferenceError. Now derived from `submissions.filter(s => !s.approved_at && !s.deleted_at).length`.
2. **MIME sanitiser** ‚Äî Anthropic Vision rejects anything except jpeg/png/gif/webp. Was sending raw content-type header (`image/heic`, `video/mp4`, etc.) and getting 400s swallowed by best-effort. Normalised at fetch.
3. **Missing return in single-file `.then()` chain** ‚Äî race condition where `describe` ran before `identify-actor` finished writing the creator field. Replaced with sequential `async` IIFE.
4. **Videos silently no-op** ‚Äî uploaded videos had no `thumbnail_url`, so identify-actor skipped them. Fixed with browser-side first-frame capture.
5. **Images had no canonical_name** ‚Äî `creative-library-describe` skipped rows with no transcript, so image rows never got renamed. Now reads `visual_description` as fallback input.
6. **Errors silently swallowed** ‚Äî `supabase.functions.invoke` doesn't throw on 4xx/5xx (it returns `{ data, error }`). Now per-file progress surfaces error messages.
7. **Unlabelled reference images** ‚Äî Claude was getting N unlabelled images preceded by a comma-joined name list (position match). Now each reference has a `Reference: NAME` text block immediately before its image.

## Known gaps / open items

- **Manually-uploaded videos >500 MB land without a thumbnail.** `captureVideoThumbnail` self-skips above that size to avoid the moov-atom stall. A backfill job could extract thumbnails server-side via ffmpeg.
- **Anthropic API HEIC support unverified.** Sanitiser maps unknown content-types to `image/jpeg` which works for the bytes themselves, but if a thumbnail's actual bytes are HEIC the API may still reject. Unlikely path since `captureVideoThumbnail` always outputs JPEG and images uploaded directly are typed by the bucket.
- **Reference set quality.** `creator` is freeform TEXT. If existing rows have mixed casing or trailing garbage ("Adam C", "adam carter"), each spawns a separate reference key. Recommend a one-off `UPDATE lib_creative_library SET creator = UPPER(creator) WHERE creator IS NOT NULL` audit before relying on the reference set at scale.
- **`onSaved` prop on `UploadModal`** is no longer called (the modal closes immediately and refresh happens via `upload-queue-idle` event). Prop left in place for now but is dead. Cleanup eventually.
