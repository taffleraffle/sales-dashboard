**Key facts:**
- No reliable confirmation signal exists in data (GHL `appointment_status` is the uniform `confirmed` default; `outcome` is 99% null). So confirmation is a **manual mark**.
- Ben's team already colour-codes calls red/green/yellow in Google Calendar.
- Show rate today is an **aggregate** (daily lives ÷ bookings from EOD). No per-booking show link — but the EOD already records attendance as the call **outcome**.

**Final design (migration 160 supersedes 159):**
- Table **`booking_call_status`** keyed on **`ghl_contact_id`** (the prospect) — so BOTH surfaces write one shared record. Holds `confirmation` ('confirmed'|'unconfirmed'|unset) only.
- **Attendance is NOT marked** — derived from `closer_calls.outcome` (closed/not_closed/showed → showed; no_show → no-show) in the rollup view.
- View **`lib_booking_confirmation_daily`** (date × audience) feeds the tiles.

**Two entry points to mark confirmation:**
1. **Marketing booking drilldowns** (`MarketingPerformance.jsx` `CallStatusCell`) — Conf / Unconf green/red toggle on Bookings / Q.Book / Cost-per-Booking / Cost-per-Q.Book rows.
2. **Closer EOD** (`EODReview.jsx`) — per-call "Confirmed? / Not confirmed" control under the outcome buttons; resolves each call's contact via `ghl_event_id → ghl_appointments`; writes the shared table immediately.

**Tiles:** "Conf. Show%" and "Unconf. Show%" in MarketingPerformance's **Calls & Show Rates** section (show% = showed ÷ (showed + no_show) per cohort; needs a logged EOD outcome to count in the rate). Read 0% until calls are marked + EODs filed.

## 4. Ad-creative attribution investigation ⚠️ ROOT CAUSE FOUND, FIX OPEN

**Trigger:** Ben asked which ad `120251079309470530` was, then which creatives Mark Thurber and Ron Adi came from.

### Findings
- `120251079309470530` = the Meta **ad *set*** named **"Adv +"** (6 creatives, all "Electricians: Book Out Your Crews" concept). Images saved to `Downloads/ad_1.png … ad_7.png`:
  - **1** — "Google Rewrote the SEO Rules. Your Agency Didn't." (cream comparison)
  - **2** — "Think Business Is Just Slow This Year? It Might Be Google." (crew + van)
  - **3 - ST** — "Google Ranks Electricians Differently Now." (serif, struck list)
  - **4** — "Your Vans Aren't Sitting Because There's No Work…" (parked vans)
  - **5** — "Your Crews Aren't Half-Busy Because Demand Dropped…" (Prime Electrical team)
  - **7** — "If you own an Electrical Company doing $1m+…" (plain green card)

- **Mark Thurber → Ad "2"** ✅ (confirmed). His true ad id `120251079309400530` is captured in his typeform submission's hidden **`h_ad_id`** field. It's NOT in the `ad_id` column because that Advantage+ ad isn't synced into our `ads` table.
- **Ron Adi → NOT recoverable.** Checked all four sources: not in `typeform_responses`, no ad id/fbclid in GHL (Zapier-fed, `medium: zapier`), nothing named "ABC" in Meta, empty in Hyros. His path dropped the ad id at every step. **"ABC – Copy" is a hardcoded UTM placeholder, not an ID.**

### Why ad IDs go missing (root cause)
1. **Funnel URL tracking template is misconfigured:**
   - `utm_term` = `{{adset.id}}` (ad **set** id) — should be `{{ad.id}}`
   - `utm_content` = hardcoded **"ABC"** — should be `{{ad.name}}` (it only becomes "ABC – Copy" when a campaign/adset is *duplicated*)
   - The real ad id only lands in a hidden `h_ad_id` field
2. **Two ingestion paths disagree:** typeform→Supabase keeps `h_ad_id` (Mark recoverable); typeform→**Zapier→GHL** drops it (Ron lost).
3. **Our `ad_id` resolver** only fills when it can match the synced `ads` table — only **19 of 159** recent click-ids are in that table (Advantage+ ads not synced), so many leads go null or name-match fallback. NB: the `ad_id` column and `h_ad_id` usually resolve to the same ad **name** but different ids (Meta spins a new ad-object id per placement).

### 🟢 Big find: **Hyros already has exact per-lead → ad attribution**
Hyros (`VITE_HYROS_API_KEY`) maps **email → exact ad name + adSourceId** for most leads, and it distinguishes "4" from "4 – Copy". It is **not wired into the dashboard**. Examples pulled live:
| Lead | Ad | adSourceId |
|---|---|---|
| john@american-elec.com | EV-10 | 120250993152170530 |
| rafael (hgservice) | RV-14 | 120250993152270530 |
| robert redd | 3 | 120249848060210530 |
| rickeydunbar / sammy | 4 – Copy | 120250635749720530 |
| hayden | RI-1 | 120250792182700530 |

Hyros API that works: `GET https://api.hyros.com/v1/api/v1.0/leads?emails=<email>` with header `API-Key: <key>` (the `phoneNumbers=` filter appears to be ignored — returns recent leads regardless).

### Recommended fix (OPEN — awaiting go-ahead)
1. **Wire Hyros → dashboard** (email → ad id/name into `typeform_responses` or a new table) as the primary attribution source. Highest leverage — gives exact creative on nearly every lead.
2. **Fix the funnel template** (`opt-funnel` repo): `utm_content={{ad.name}}`, `utm_term={{ad.id}}` (or a dedicated `ad_id={{ad.id}}`), and pass the ad id through the Zapier→GHL zap. Then typeform AND GHL leads carry the exact ad natively.
3. **Map `h_ad_id → ad_id`** in ingestion (also backfills history, incl. Mark).
4. **Sync Advantage+ ads** into the `ads` table so ids resolve to creatives.
