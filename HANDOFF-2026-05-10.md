## Three real gaps left (in priority order)

### 1. UTM → setter_leads attribution (~1 hour)
**Why:** every gallery card shows `Untested` KPI badge because we never join
`utm_content` from a setter_lead back to its originating ad. Once wired:
- Cards show `In KPI / Marginal / Out of KPI` based on real cost-per-booked / cost-per-close / lead-quality %
- The `derive_variant_states()` function gets real data to classify against
- The analyst can answer "which ads are bringing closeable leads?"

**How:**
- Verify `setter_leads.utm_content` is being populated from GHL on lead webhook
- If yes: add a SQL function that joins `setter_leads.utm_content = ads.ad_id` and aggregates `closer_calls.outcome` per ad
- Update `AdsGallery.jsx`'s data load to call that function and populate `stats.booked / closed / leadQualityPct`

### 2. Variant tagging (~2-4 hours either path)
**Why:** the parser from migration 012 only matches SOP names like
`H4.2_BA-PROOF_S-OFFICE_OSO_v1`. Current ad names are like `Image - Drawing
- fully booked - Copy`. So `variant_id` is null on every ad → state machine
returns 0/0 → phrase scoring stays at ad-level (works, just not aggregated
up to variant level).

**Two paths:**
- **A. Adopt SOP** — rename ads in Meta Ads Manager going forward to follow
  the SOP. Old ads stay un-tagged, new ones auto-tag on next sync.
- **B. Manual tag UI** — build a 4-component picker (hook / body / scene /
  creator) on the gallery card. Operator clicks "Tag", picks values, save
  inserts a `library.variants` row + updates `public.ads.variant_id`. Phase
  3 of `AD-PERFORMANCE-PLAN.md` already specs this.

Path B is more work but works for legacy ads. Recommend B if backfilling
the 381 existing ads matters.

### 3. Per-page editorial polish (~3-4 hours)
**Why:** all 22 pages have editorial **headers** but their **internal
panels** still inherit only the palette + sharp corners. The full editorial
treatment (eyebrow labels per section, "what this means" callouts after
data sections, serif tabular numbers everywhere, mobile responsiveness
verified on real device) hasn't been done page-by-page.

Highest-value first: SalesOverview (already mostly done), Marketing
Performance, Closer/Setter detail.

genuinely new signal layer (what the creator says on camera vs the ad copy
text). Path:
1. Add `OPENAI_API_KEY=sk-proj-...` to `.env` (or Render env)
2. Paste this SQL to allow anon writes on `library.creative_transcripts`:
   ```sql
   BEGIN;
   DROP POLICY IF EXISTS "anon write transcripts" ON library.creative_transcripts;
   CREATE POLICY "anon write transcripts" ON library.creative_transcripts
     FOR ALL TO anon, authenticated USING (true) WITH CHECK (true);
   NOTIFY pgrst, 'reload schema';
   COMMIT;
   ```
3. Run `node scripts/transcribe-ad-videos.mjs 5` to test on 5 videos (~30 cents)
4. Once verified, re-run uncapped: `node scripts/transcribe-ad-videos.mjs 1000`
